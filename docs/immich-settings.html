01 / State

Installed, but deliberately stopped

Flux still owns the release, Services, Ingress, PVCs, configuration, and database. Only the three application controllers are scaled down.

Immich is unavailable right now.

Server, machine learning, and Valkey each have replicas: 0. The Tailscale address exists, but there is no Immich server pod behind it.

Server replicas

server.controllers.main.replicas
0 · OFF

Controls the API, web UI, uploads, library jobs, metadata extraction, thumbnails, and video handling.

Set to 1: Immich becomes reachable and background work resumes. Set to 0: no UI, API, uploads, or jobs.

Machine-learning replicas

machine-learning.controllers.main.replicas
0 · OFF

Runs facial recognition, smart search, and OCR inference through the RTX 3060.

Set to 1: ML jobs run. Set to 0: the server can run, but ML-backed jobs fail or remain queued.

Valkey replicas

valkey.controllers.main.replicas
0 · OFF

Provides Immich's Redis-compatible queue and cache service.

Set to 1: job queues and cache work. Set to 0: the Immich server cannot operate correctly even if its own replica is enabled.

PostgreSQL instances

Cluster.spec.instances
1 · ON

CloudNativePG keeps the application database online independently of the paused Helm workloads.

Current outcome: metadata remains available and no database bootstrap is needed when Immich resumes. This is a single instance, not high availability.

CPU is intentionally not capped.

Background imports can use spare CPU and finish sooner. The measured server plus ML load was about 1.3 cores while face detection and indexing were active—not runaway use on this node.

04 / Features

Application behavior

Video transcoding

immich.configuration.ffmpeg.transcode
disabled

Immich will not create converted video or audio streams, saving CPU, GPU time, and encoded-video storage.

Trade-off: source formats unsupported by a browser or client may not play, and high-bitrate originals use more network bandwidth.

NVENC acceleration

ffmpeg.accel / ffmpeg.accelDecode
nvenc / true

NVIDIA encode and hardware decode are configured, and the server uses the nvidia runtime class.

Current outcome: effectively dormant because transcoding is disabled. If transcoding is re-enabled, video work uses the RTX 3060.

Machine learning

machine-learning.enabled
true

The ML Deployment, Service, cache PVC, CUDA image, and GPU request are rendered by Helm.

Important: enabled means “resources exist”; its replica count of zero means the pod is currently stopped.

Config ownership

immich.configurationKind
ConfigMap

Settings are generated from this repository and mounted as Immich's configuration file.

Outcome: settings shown as config-file-controlled in the admin UI must be changed in Git, not in the UI.

Trash

immich.configuration.trash
chart default

The repository contains only commented examples, so Immich's default trash behavior applies.

If explicitly disabled: deletions bypass trash. If enabled: deleted items remain recoverable for the configured number of days.

Storage template

immich.configuration.storageTemplate
chart default

The example template is commented out; Immich's default organization is used for managed uploads.

If enabled later: new managed assets are moved into the declared year/month/day filename layout.

06 / Data

One app-specific CloudNativePG database

PostgreSQL 18

immich-database · instances: 1
running

A single CNPG instance owns database immich and user immich. Credentials come from the operator-generated immich-database-app Secret.

Outcome: simple single-node persistence. A node/disk failure has no live database replica to fail over to.

Vector support

vchord + vector + cube + earthdistance
enabled

VectorChord is preloaded and Immich's required extensions are ensured by CNPG.

Outcome: semantic search and facial-recognition embeddings can be stored and queried.

Database backup CronJob

schedule: 0 2 * * * · Asia/Kolkata
suspended

When enabled, it writes compressed SQL dumps to /home/bupd/hdd/data/immich/backups, keeps today and yesterday, plus the newest Sunday checkpoint, and removes older matching dumps.

Current outcome: no scheduled database backups run. Changing suspend to false enables the 02:00 daily schedule. This backs up database metadata only, not photos.

10 / Operations

What to change when resuming Immich

Set these three replica values to one together, then publish the artifact. Keeping the dependency set coherent avoids a server pod starting without its queue or ML endpoint.

valkey:
  controllers:
    main:
      replicas: 1

server:
  controllers:
    main:
      replicas: 1

machine-learning:
  controllers:
    main:
      replicas: 1

Expected result after resume

The UI returns, Valkey accepts queues, and CUDA machine-learning jobs resume. Video transcoding remains disabled; database and all persistent data are reused.
